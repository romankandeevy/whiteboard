## 🐛 БАГИ — исправить в первую очередь

### 1. Утечка памяти в `onMove` (canvas.js)
В функции `onMove` на каждое движение мыши создаётся новый `canvas`:
```js
const tmp = document.createElement('canvas'); // ← КАЖДЫЙ mousemove!
```
Исправление: вынести `tmp` и `tmpCtx` как модульные переменные, создавать один раз, пересоздавать только при resize.

### 2. Дублирование функции `isLight()`
Функция `isLight(hex)` определена дважды — в `toolbar.js` (строка 13) и в `selection.js` (строка 377). Вынести в общий модуль или в `canvas.js`, остальные убрать.

### 3. `touchend` — пустой `e.touches`
В `stopDraw` вызывается `getPos(e)`, которая читает `e.touches[0]`. При `touchend` массив `touches` пуст. Нужно использовать `e.changedTouches[0]`.

### 4. Ручки resize нарисованы, но не работают
В `selection.js` функция `drawSelection` рисует 8 ручек вокруг выделенной фигуры, но mousedown на этих ручках ничего не делает. Пользователь видит интерактивный UI который обманывает. Реализовать resize через ручки или убрать их до реализации.

### 5. `shapeStart` не сбрасывается при смене инструмента
Если пользователь начал рисовать фигуру (нажал кнопку мыши) и сменил инструмент — `shapeStart` остаётся заполненным. При следующем mouseup фиксируется случайная фигура. Сбрасывать `shapeStart = null` в `setTool()`.

### 6. Текст — blur срабатывает при клике на тулбар
В `startTextInput` стоит `blur → setTimeout(commit, 100)`. Если пользователь кликает на кнопку тулбара во время редактирования текста, текст коммитится раньше чем ожидается. Нужно проверять `relatedTarget` в обработчике blur.

### 7. Undo не работает для текстового инструмента корректно
При редактировании существующего текста (`strokeIndex >= 0`) старый stroke удаляется из `strokes` но не попадает в `redoStack`. История ломается.

### 8. `resize` окна — контент смещается
При `window resize` восстановление через `off.drawImage(tmp, 0, 0)` не учитывает pixel ratio (`devicePixelRatio`). На Retina-дисплеях картинка размывается.

## ⚡ ПРОИЗВОДИТЕЛЬНОСТЬ

### Rendering pipeline
Заменить синхронные вызовы `flush()` в обработчиках событий на RAF-цикл:
```js
let dirty = false;

function scheduleRender() {
  if (!dirty) {
    dirty = true;
    requestAnimationFrame(() => {
      dirty = false;
      flush();
    });
  }
}
```
Вызывать `scheduleRender()` вместо `flush()` везде.

### Тайлинг для больших досок
При большом количестве strokes (>500) — разбить offscreen canvas на тайлы 512×512 и перерисовывать только изменённые тайлы. Или использовать `OffscreenCanvas` + Worker для рендеринга в фоне.

### Дебаунс localStorage
Сейчас `saveToStorage()` вызывается на каждое событие. При активном рисовании это сотни записей в секунду. Заменить на debounce с задержкой 500ms.

### Кэширование сложных фигур
Для `drawStar` и других сложных фигур — кэшировать path в `Path2D` объекте при первой отрисовке.

### `devicePixelRatio` поддержка
Реализовать правильный Retina/HiDPI рендеринг:
```js
const dpr = window.devicePixelRatio || 1;
canvas.width = window.innerWidth * dpr;
canvas.height = window.innerHeight * dpr;
canvas.style.width = window.innerWidth + 'px';
canvas.style.height = window.innerHeight + 'px';
ctx.scale(dpr, dpr);
```
